When people say "notations in DSA", they are usually referring to asymptotic notations, which are used to describe the efficiency of an algorithm.
      The purpose is:
                To measure how an algorithm's running time or memory usage grows as the input size grows.
            We don't measure actual seconds because that depends on:
                CPU speed
                RAM
                Programming language
                Operating System
            Instead, we measure growth rate.

Why do we need notations?
          Suppose two algorithms search for an element.
          Algorithm A
                for(int i = 0; i < n; i++) {
                    if(arr[i] == target)
                        return i;
                }
          Algorithm B
                return map.get(target);
          For:
                n = 10
          both seem fast.
          For:
                n = 10,000,000
            Algorithm B is dramatically faster.
            We need a standard way to express this difference.
            That's where asymptotic notations come in.

The Three Main Notations
          1. Big O Notation - O()
                Most important notation.
                    Represents:
                          Upper bound (Worst Case)
                    Meaning:
                          At worst, how much time can this algorithm take?

              Example
                  Linear Search:
                    for(int i = 0; i < n; i++) {
                        if(arr[i] == target)
                            return i;
                    }
                  Worst case:
                        Target at end
                            or
                        Target not found
                  Checks all elements.
                        n operations
                  So:
                        O(n)

              Example
                   Array access:
                        arr[50]
                   Always one operation.
                         O(1)
                    (Constant Time)

            2. Omega Notation - Ω()
                    Represents:
                        Lower bound (Best Case)
                    Meaning:
                        At minimum, how much time can this algorithm take?
                    Linear Search
                        [10,20,30,40]
                    Find:
                        10
                    Found immediately.
                    Only one comparison.
                        Ω(1)

            3. Theta Notation - Θ()
                    Represents:
                        Exact Bound
                    Meaning:
                        Best and worst cases grow at the same rate.
                    Example
                        Print all elements
                        for(int i = 0; i < n; i++) {
                            System.out.println(arr[i]);
                        }
                    No matter what:
                        n operations
                    Best case:
                        Ω(n)
                    Worst case:
                        O(n)
                    Therefore:
                        Θ(n)

Common Time Complexities
        O(1) - Constant
            arr[5]
              n = 10      → 1 operation
              n = 1000    → 1 operation
              n = 1M      → 1 operation
        Excellent.

        O(log n) - Logarithmic
            Example:
                Binary Search
                    1000 elements
                      1000
                      500
                      250
                      125
                      ...
                Keep dividing by 2.
                Very efficient.

        O(n)
            Example:
              Linear Search
                  for(int i=0;i<n;i++)
              Input doubles:
                  100 → 200
              Work doubles.

        O(n log n)
            Efficient sorting algorithms:
                    Merge Sort
                    Heap Sort
                    Quick Sort (average)
            Examples:
                    1000 elements
                    ≈ 1000 × log(1000)
            Very good.

        O(n²)
            Nested loops.
                for(int i=0;i<n;i++) {
                    for(int j=0;j<n;j++) {
                    }
                }
            Operations:
                n × n
            For:
                1000 elements
              ≈ 1,000,000 operations.
            Can become slow.

         O(2ⁿ)
              Exponential.
          Example:
              Some recursive brute-force solutions.
                  n = 10 → 1024
                  n = 20 → 1,048,576
              Very bad growth.

         O(n!)
                Factorial.
            Example:
                Generating all permutations.
                    n = 10
                    10! = 3,628,800
                Terrible for large inputs.

Complexity Ranking
      Best → Worst
            O(1)
            ↓
            O(log n)
            ↓
            O(n)
            ↓
            O(n log n)
            ↓
            O(n²)
            ↓
            O(n³)
            ↓
            O(2ⁿ)
            ↓
            O(n!)

Space Complexity
      Notations are also used for memory.
      Example
          int sum = 0;
      Extra memory:
          O(1)
  
      Example
          int[] temp = new int[n];
      Extra memory:
            O(n)
      because memory grows with input size.

Real Software Examples
      HashMap Lookup
            map.get(key);
      Average:
          O(1)

      ArrayList Access
            list.get(index);
          O(1)
      
      ArrayList Search
            list.contains(value);
          O(n)
