What is Proxy Pattern?
    The Proxy Pattern is a Structural Design Pattern.
      The main idea is:
            A Proxy object acts as a substitute (middleman) for a real object and controls access to it.
          Think of it like a security guard standing in front of a building.
              You don't directly enter the building.
              You first talk to the guard.
              The guard may:
                    Allow access
                    Deny access
                    Log your visit
                    Delay creation of the real object until needed

Real-world Example
    Imagine watching a YouTube video.
    You don't directly communicate with the video storage servers.
    Instead:
          User
            ↓
          YouTube Proxy/CDN
            ↓
          Video Server
    The proxy can:
          Cache videos
          Check permissions
          Reduce network calls

    Structure
                Client
                   |
                   v
              Subject Interface
                   ^
                   |
           -------------------
           |                 |
           |                 |
         RealSubject      Proxy
    Both the Real Object and Proxy implement the same interface.

Example: Image Viewer
        Suppose loading an image from disk is expensive.
          
          Step 1: Subject Interface
                    public interface Image {
                        void display();
                    }

          Step 2: Real Object
            This object does the heavy work.
                    public class RealImage implements Image {
                        private String fileName;
                        public RealImage(String fileName) {
                            this.fileName = fileName;
                            loadFromDisk();
                        }
                        private void loadFromDisk() {
                            System.out.println("Loading image from disk: " + fileName);
                        }
                        @Override
                        public void display() {
                            System.out.println("Displaying image: " + fileName);
                        }
                    }

          Step 3: Proxy
                  public class ProxyImage implements Image {
                        private String fileName;
                        private RealImage realImage;
                        public ProxyImage(String fileName) {
                            this.fileName = fileName;
                        }
                        @Override
                        public void display() {
                            if (realImage == null) {
                                realImage = new RealImage(fileName);
                            }
                            realImage.display();
                        }
                }
                Notice:
                      private RealImage realImage;
                Initially:
                      realImage = null;
                The actual image is created only when needed.

          Step 4: Client
                public class Main {
                    public static void main(String[] args) {
                        Image image = new ProxyImage("profile.jpg");
                        System.out.println("Proxy created");
                        image.display();
                        image.display();
                    }
                }

Output
        Proxy created
            Loading image from disk: profile.jpg
            Displaying image: profile.jpg
        Displaying image: profile.jpg
        Notice:
            First call:
                        image.display();
                    creates:
                        new RealImage(...)
                    and loads the image.
            Second call:
                    image.display();
                doesn't load again.
            It reuses the already created object.

Why not directly use RealImage?
        Without Proxy:
            Image image = new RealImage("profile.jpg");
        Output:
            Loading image from disk: profile.jpg
        happens immediately, even if you never call:
                image.display();
            That's wasteful.
        The proxy delays object creation until it is actually needed.
        This is called Lazy Initialization.

Another Example: Security Proxy
        Suppose only ADMIN users can delete files.
        Interface
            public interface FileService {
                void deleteFile(String fileName);
            }

        Real Service
            public class RealFileService implements FileService {
                @Override
                public void deleteFile(String fileName) {
                    System.out.println(fileName + " deleted");
                }
            }

        Proxy
            public class FileServiceProxy implements FileService {
                    private String role;
                    private RealFileService realService;
                
                    public FileServiceProxy(String role) {
                        this.role = role;
                        this.realService = new RealFileService();
                    }
                
                    @Override
                    public void deleteFile(String fileName) {
                
                        if ("ADMIN".equals(role)) {
                            realService.deleteFile(fileName);
                        } else {
                            System.out.println("Access Denied");
                        }
                    }
                }

        Client
                public class Main {
                    public static void main(String[] args) {
                        FileService service = new FileServiceProxy("USER");
                        service.deleteFile("data.txt");
                    }
                }

        Output:
                Access Denied
                The proxy controls access before reaching the real object.
        
        Types of Proxy
            1. Virtual Proxy
                    Creates expensive objects only when needed.
                Example:
                    ProxyImage
            2. Protection Proxy
                    Controls access.
                Example:
                    FileServiceProxy
                    Checks permissions.
            3. Remote Proxy
                    Represents an object located on another machine.
                        Client
                          ↓
                        Remote Proxy
                          ↓
                        Remote Server
                    Common in distributed systems and RPC frameworks.
            4. Caching Proxy
                    Stores results to avoid repeated expensive operations.
                Example:
                    getUser(1)
                First call:
                    Database queried
                Second call:
                    Returned from cache

So the proxy patterns can be used to,
        delay creation
        check permissions
        cache data
        log actions
    before forwarding the call.

One liner explanation:
        The Proxy Pattern provides a placeholder or surrogate object that controls access to a real object. The proxy implements the same interface as the real object and can perform additional actions such as lazy loading, security checks, caching, logging, or remote communication before delegating the request to the real object.
